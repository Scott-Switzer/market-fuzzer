Product: what it does, and what it does not

Strategy breaker → Evidence exporter. Not a backtester pretending to be a validator.

MVP scope

  • Plain-English strategy interpretation → clause ledger
  • Immutable locked strategy version with canonical hash
  • Historical multi-asset backtest with realistic costs
  • Sealed synthetic campaign with hidden world families
  • Failure predicates, minimization, replay, evidence package
  • Quant validation APIs: CPCV, SPA/MCS, deflated Sharpe, PBO bounds
  • Deterministic exchange with TCA and toxicity metrics
  • Playwright browser E2E with persisted screenshots

Out of scope

  • No brokerage connection or live trading
  • No proprietary data subscription required
  • No arbitrary Python execution
  • No alpha or profitability guarantees
  • No hidden-parameter leakage in public reports
  • No universal realism claim for synthetic worlds

Claim boundary is explicit: reproducible counterfactual evidence within declared mechanisms, not forecast or live-trading approval.

Moat and differentiation

The hard parts are governance and boundaries, not rendering.

Moat sources

  • Sealed evaluation lifecycle — public and hidden scores separated by design, not by convention.
  • Deterministic evidence ledger — byte-equivalent reproducibility claims with hashes and audit events.
  • Open-core boundary — deterministic engine MIT; enterprise extensions as optional add-ons.
  • Anti-memorization universe — ProceduralUniverseGenerator; 1000+ assets in ~0.14s; rotation/session determinism.
  • External adapter boundary — data vendors can certify feeds without changing internal evaluation.

Where to win

  • Solo quants / 2–20 person firms — “I have Python and Jupyter, not a model-validation stack.”
  • Quant labs + bootcamps — deterministic instructor challenge with hidden reveal.
  • Post-backtest validation layer — teams already using QC/QuantRocket pipe a finalized strategy here.
Universe scale. Baseline presets + Fenrix 8 + synthetic_large 220 + procedural 1000+.
